# راهنمای قابلیت دائمی اصلاح نتیجه مسابقه

## چه چیزی اضافه شده است؟

در بخش «مسابقات پایان‌یافته» دکمه «اصلاح نتیجه ثبت شده» اکنون یک پنجره اختصاصی باز می‌کند. مدیر نتیجه صحیح و علت اصلاح را وارد می‌کند و پس از تأیید:

1. نتیجه قبلی و جدید در جدول `match_result_revisions` ذخیره می‌شود.
2. پیش‌بینی‌های کاربران بدون تغییر باقی می‌ماند.
3. امتیاز و رتبه همه کاربران از روی نتایج فعلی دوباره محاسبه می‌شود.
4. جزئیات عملیات در `audit_logs` ثبت می‌شود.
5. همه تغییرات در یک تراکنش دیتابیس انجام می‌شود؛ در صورت خطا، عملیات کامل بازگردانده می‌شود.
6. تاریخچه اصلاحات همان مسابقه در پنجره ویرایش قابل مشاهده است.

جدول جدید در اولین اجرای نسخه جدید، به‌صورت خودکار ساخته می‌شود و نیازی به اجرای migration دستی نیست.

## فایل‌های اصلی تغییرکرده

- `database.py`
- `main.py`
- `templates/admin.html`
- `requirements.txt` فقط از UTF-16 به UTF-8 تبدیل شده تا نصب وابستگی‌ها در سرور پایدارتر باشد.

## تنظیمات پیشنهادی در Liara

در بخش Environment Variables برنامه، این متغیرها را تعریف کنید:

```env
ADMIN_USERNAME=admin
ADMIN_PASSWORD=یک-رمز-قوی-و-جدید
ADMIN_SESSION_SECRET=یک-رشته-تصادفی-طولانی-حداقل-۳۲-کاراکتر
ADMIN_COOKIE_SECURE=true
```

برای ساخت `ADMIN_SESSION_SECRET` می‌توانید محلی اجرا کنید:

```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

پس از استقرار نسخه جدید، یک‌بار از پنل خارج شوید و دوباره با حساب مدیر وارد شوید تا نشست امن مدیریتی ایجاد شود.

## روش استقرار

1. پیش از استقرار، از دیسک `data` و فایل `data/football.db` نسخه پشتیبان بگیرید.
2. فایل‌های تغییرکرده را در مخزن GitHub جایگزین و Commit کنید.
3. متغیرهای محیطی بالا را در Liara ثبت کنید.
4. نسخه جدید را Deploy کنید.
5. وارد پنل ادمین شوید و از بخش مسابقات پایان‌یافته، نتیجه موردنظر را اصلاح کنید.
6. جدول رده‌بندی و تاریخچه Logs را بررسی کنید.

## رفتار مهم

- اصلاح نتیجه فقط برای مسابقه‌ای مجاز است که وضعیت آن `finished` باشد.
- گل‌ها باید عدد صحیح بین صفر تا ۹۹ باشند.
- واردکردن علت اصلاح الزامی است.
- ثبت نتیجه مشابه نتیجه فعلی رد می‌شود.
- این عملیات پیام «سوت پایان» جدیدی به کانال بله ارسال نمی‌کند تا اعلان تکراری منتشر نشود.

## نکته امنیتی مهم

در نسخه فعلی پروژه، چند کلید سرویس و رمز در کد اصلی قرار گرفته‌اند. چون این فایل‌ها در مخزن GitHub بوده‌اند، بهتر است آن کلیدها را باطل و دوباره صادر کنید و سپس آن‌ها را نیز به Environment Variables لیارا انتقال دهید.
